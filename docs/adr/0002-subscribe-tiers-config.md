# ADR 0002: 认购档位可配置（仅档位，不做管理代认购）

## 状态

已接受（2026-07-13）

## 背景

用户端节点页原先写死认购金额档位；运营需要随时调整快捷金额，且希望用户可自填不低于最低认购额的任意金额。曾讨论是否一并做「管理端代用户认购」。

## 决策

选 **A：仅档位配置 + 用户自填**，不做管理代认购。

- 存储：`business_configs.subscribe_tiers`（逗号分隔，如 `100,500,1000,3000`）
- 公开读：`GET /api/app_server/subscribe_tiers` → `{ tiers, min_subscribe_amount }`（无需登录）
- 管理改：既有业务参数页改该键；热更新后即时生效
- 下单：仍 `POST /api/app_server/buy`；金额 ≥ `min_subscribe_amount`
- 档位与出局倍数档位分离：快捷金额 ≠ 倍数区间定义

## 后果

- _positive_：运营可改快捷档位，用户端无需发版写死金额
- _positive_：自填路径复用既有 buy 校验，无新订单通道
- _negative_：管理端不能替用户扣余额下单；若以后需要代购须另开决策
