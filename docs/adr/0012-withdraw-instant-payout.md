# ADR 0012: 提取确认即掉头并异步打款（取消审核）

## 状态

已接受（2026-07-17）

## 背景

原流程：用户申请 → `pending` 待审 → 管理端通过时利率掉头并进 `rewarded` → 再链上打款。产品改为确认即生效，去掉审核。

## 决策

1. 用户确认提取时（`CreateWithdraw`）：校验打款已开启且配置齐全；扣可提；建单直接为 `rewarded`；对全部进行中认购单做利率掉头（语义见 ADR 0018）；写提取流水；**异步**调用打款（失败不回滚可提）。
2. 打款关闭或未配置时：**拒绝**提取（不扣、不掉头）。
3. 取消用户取消、管理端通过/拒绝；管理端保留打款重试/确认回执。
4. 启动时一次性将遗留 `pending` 全额退回可提并标 `cancelled`。

## 后果

- _positive_：出金路径更短，掉头与确认对齐用户动作
- _negative_：误点无法撤销；强依赖生产打款开关与热钱包；异步打款失败需运维重试
- 部分废止 ADR 0010 中「审核与打款分离」的审核段；打款幂等与状态字仍沿用 0010
