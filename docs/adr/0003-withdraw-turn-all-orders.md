# ADR 0003: 提取通过后掉头全部进行中认购订单

## 状态

已接受（2026-07-13）

## 背景

曾约定提取须由用户指定要掉头的认购订单。产品改为：用户只填金额，通过后对该用户**全部仍进行中**的认购订单掉头。

## 决策

- 用户端不提供订单多选
- 管理端**通过当下**再查该用户全部 `active` 认购订单并掉头（非申请快照）
- 已出局订单不参与
- **允许**无进行中认购时申请提取；通过时若仍无进行中订单，则只确认提取、不掉头

## 后果

- _positive_：前端更简单，与「提取即影响整户静态节奏」的运营口径一致
- _negative_：无法只掉头部分订单；多笔进行中时一次提取影响面更大
