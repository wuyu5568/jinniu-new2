# 金牛协议（认购与分红）

用户以认购金额开立认购订单，按日获得静态收益；推荐与社区激励产生每日分红。两类收益均可推进订单出局进度，但只有认购金额计入本金。

**MVP（第一期）**：四期交付中的第 1 期——用户（钱包 eth_authorize 登录；首次须填邀请码）、认购下单、订单级静态收益（含提取掉头）、可提余额与 6% 提取；管理 API 与用户 API 同仓库同进程、分目录。第 2 期代数分红；第 3 期社区 V 级 / 级差 / 平级 / 断档；第 4 期对接既有 **dapp-admin**；链上充值见 ADR 0007；**提取链上打款**见 ADR 0010（默认关，开后热钱包打到账额）。

## Language

**认购金额**:
用户开立认购订单时投入的本金（U）。出局倍数按该金额所在档位计算。
_Avoid_: 质押金额、投资额、本金（对外口径）

**认购订单**:
一笔认购形成的订单，记录认购金额、出局倍数、已累计收益与状态（进行中/已出局）。代码与持久化沿用标识 Location / locations，语义等同本词。
_Avoid_: 质押订单、仓位

**静态收益**:
认购订单按当日静态收益率释放的收益，计算公式为 **出局金额 × 静态收益率**（出局金额 = 认购金额 × 出局倍数）；**双记**：增加该单出局累计，并增加用户可提余额。
_Avoid_: 日息、自然释放、挖矿收益（对外口径）；按认购金额计息

**静态收益率**:
挂在**认购订单**上：每笔订单开单第 1 天为 0.6%，之后在 0.6%–1.4% 间按日步进 0.05% 升降；与同用户其他订单相互独立。
_Avoid_: 日利率、释放比例、用户级统一利率（以及错误写法「0.5% 步进」）

**每日分红**:
推荐/社区激励产生的收益类型统称。**代数奖励、社区基础奖、平级分红**按 FIFO 填入本人进行中认购单：增加出局累计，并同步增加可提余额（双记）；无进行中订单则当日该笔作废，填满全部进行中单后的剩余亦作废（不作废部分以外的金额不进可提）。
_Avoid_: 动态收益；无单仍发可提；超额部分仍进可提

**账户余额**:
用户可用于认购的中心化账本余额；来源含：管理端充值（增加）、管理端修改账户余额（设目标值）、以及链上充值——用户通过 **BSC 主网充值合约**（BuySomething，`0x49c735…`）入账；管理端 HTTP 触发按合约序号逐条拉取；**≥100 USDT** 且已注册才入账；未注册或低于最低额仍推进游标并记跳过（`eth_user_record` 一序号一行，`UNIQUE(last)`）；补注册后可按序号重放 skipped 入账。与可提余额分开记账。
_Avoid_: 钱包余额、可提余额；仅创世 JWT 充值；每用户独立充值地址；USDT Transfer 扫块 / eth_getLogs；Webhook/独立扫块进程；未注册卡住后续序号；`<100` 中断整批拉取；无唯一游标导致重复入账；跳过不推进游标；已跳过序号靠普通拉取自动补账

**可提余额**:
用户当前可提取的资产余额；静态收益与每日分红（代数/社区/平级）入账时增加（分红仅就可填进出局空间的部分），提取资产时扣减；管理端可修改为目标值（差额记管理端调整可提流水）。不可划回账户余额；再认购须重新充值账户余额。
_Avoid_: 钱包余额（链上含义）、账户余额（认购用）；可提划转认购；无单/超额分红仍进可提

**管理端调整可提**:
管理端将可提余额设为目标值时产生的账本流水类型；金额为差额（可正可负）。不计入用户收益、不进入账户充值列表。
_Avoid_: 充值、提取、静态收益

**出局**:
认购订单的累计入账（静态收益 + 每日分红）从 0 起加总，达到「认购金额 × 出局倍数」时订单结束、停止继续释放。
_Avoid_: 退出、清仓、爆仓；开单时预记认购金额再累加

**出局倍数**:
由认购金额档位决定：100–999U 为 2 倍，1000–2999U 为 2.5 倍，≥3000U 为 3 倍。
_Avoid_: ROI、退出倍数
    
**提取资产**:
用户从可提余额申请提出指定金额。金额不得低于业务参数「最低提取金额」（默认 10）。申请时立刻扣减可提、按当时费率锁定手续费（默认到账 94%）；订单进入待审（`pending`）。**不要求**申请时已有进行中认购订单。管理端通过时：账本确认 + **按通过当下**对该用户全部仍进行中的认购订单做次日静态收益率掉头（若当时没有进行中订单则只确认、不掉头），状态变为 **待打款（`rewarded`）**。链上打款与审核分离：热钱包将到账额打至用户地址后状态为 `pass`；打款中为 `doing`。拒绝或用户取消则全额退回可提、不掉头、不打款。拒绝原因可选。允许多笔待审。用户端无需自选掉头订单。代码/RPC/表贯穿标识 Withdraw / withdraws。
_Avoid_: 提现（可作同义，优先用提取资产）；用户自选部分订单掉头；按申请快照掉头；无进行中订单禁止提取；免手续费；申请即掉头；通过即视为已到账；打款失败自动退可提；无 tx_hash 保护下重复广播；把 Withdraw 理解成与审核同步的自动转账；对已出局订单掉头

**提取打款**:
将已审待打（`rewarded`）的提取单，由服务端热钱包在 BSC 上向用户地址转 USDT（金额为到账额）；广播后先记交易哈希再确认上链，成功后为 `pass`。可由定时任务与管理端触发。详见 ADR 0010。
_Avoid_: 审核通过同步强制上链且无独立状态；无私钥管控的多实例并行打同一单

**用户**:
以钱包地址为身份标识；登录时对**地址本身**做 personal_sign（无 nonce）；服务端验签后签发 JWT。首次登录须在同一次登录请求中提交邀请码并绑定邀请人（创世账号除外）。
_Avoid_: 用户名密码登录；先验签再另接口绑邀请码；本期强制 nonce 挑战（已有 v1 接口保留，兼容前端 eth_authorize 走无 nonce）

**激活用户**:
至少开立过一笔认购订单的用户（含已出局）。看板 `totalUser` = 激活用户数；`todayUser` = 当日首次开立认购订单的用户数（上海自然日）。注册未认购不算激活。
_Avoid_: 未锁定用户、未软删用户、仅进行中认购才算激活

**登录挑战**:
可选能力：v1 接口仍可下发一次性 nonce；当前用户端兼容登录（eth_authorize）不使用挑战，直接对地址签名。
_Avoid_: 把 JWT 当成登录挑战；本期前端主路径依赖 nonce

**邀请码**:
首次登录时必填；值为**已注册用户的钱包地址**。校验通过后建立与邀请人的推荐关系，方可完成注册/登录。创世账号除外。
_Avoid_: 独立邀请口令、随机短码（非本项目口径）

**邀请人**:
邀请码所对应的已注册用户；成为新用户的直属上级（推荐关系）。**绑定后永久不可更改**。
_Avoid_: 推荐人（可作同义，优先用邀请人）；可改上级

**创世账号**:
预置的根钱包用户；可不填邀请码即可完成首次登录/注册，作为推荐树顶点。地址由配置或初始化写入。
_Avoid_: 管理员（权限角色）、普通邀请人

**静态收益结算**:
对进行中认购订单按日入账静态收益并推进次日收益率；正式环境由进程内 cron 每日定时执行（可配置表达式与时区，默认上海），管理端可手动触发。同日后续顺序为：代数奖励 → 刷新社区等级 → 社区基础奖 → 平级分红。以 `settle_runs` 按上海自然日 **INSERT 占位**防重（多实例安全）：占不到则跳过；占位后中途失败不自动重跑。`force=1` 仅当配置允许（`allow_force_settle`）时可强跑（可能重复入账，仅测试）。
_Avoid_: 仅懒结算、无定时任务；先按旧等级发社区奖再刷新等级；生产环境开启 force；多实例先查再写而不占位

**管理 API**:
与用户 API 同仓库、同一服务进程，路径前缀 `/api/admin_jinniu`；管理端账号密码登录后发 JWT，用于充值、查数、触发结算等运维操作。
_Avoid_: 与用户接口混在同一巨型 proto；MVP 独立管理微服务；admin_key 明文头鉴权（已删除）

**管理 Web**:
运维界面使用既有仓库 **dapp-admin**（不拷进本仓）；覆盖充值、用户/订单、结算、邀请关系、业务参数调整等。
_Avoid_: 本期在本仓新建 Vite+React 管理端；密钥长期存 localStorage

**API 列表响应**:
列表类 HTTP 接口响应体为裸 JSON 数组 `[{...},...]`；分页 token 放在响应头 `X-Next-Page-Token`。收益按日汇总另走 `/earnings/days`。单资源（资料、充值结果、看板、结算结果等）仍为 JSON 对象。
_Avoid_: 列表再包一层 `{"users":[...]}`；把分页 token 塞进数组元素

**用户 Web**:
同仓库 `web/app`（Vite + React）；登录仅 MetaMask：连钱包 → 查是否注册 → 未注册填邀请码再签名；JWT 存 sessionStorage；规划增补下级树邀请关系、每日收益日历。
_Avoid_: 登录页私钥入口；先签名再因缺邀请码失败

**每日收益明细**:
当日入账流水，类型含静态收益、代数奖励、社区基础奖、平级分红（提取与充值不计）。
_Avoid_: 仅静态；把提取/充值算作收益

**业务参数**:
管理端以键值表（`business_configs`）逐项调整并写入 MySQL；进程内聚合为业务参数对象热更新，立即用于后续业务，不回溯历史。含：提取手续费；代数比例与最大代数；平级比例与最低 V 级；静态利率上下限与步进；最低认购/提取金额；认购档位；出局倍数三档；社区 V1–V9 门槛与比例。创世地址、JWT/管理密钥等仍在配置文件。
_Avoid_: 仅改 config.yaml；改参后重算历史；在页面改 JWT/管理密钥；以整包 JSON 为唯一编辑入口

**认购档位**:
管理端可配置的一组预设认购金额（如 100 / 500 / 1000 / 3000），供用户端快捷选择；用户亦可自填任意金额，但不得低于最低认购金额。档位本身不决定出局倍数（出局倍数仍按金额区间）。
_Avoid_: 管理端代用户认购；档位与出局倍数档位混为一谈；仅允许点选档位不可自填

**管理用户数据**:
管理端可对用户主档与认购订单做增删改查。邀请人**创建后不可改**。删除为双接口：默认软删除；硬删除须无下级、无认购订单、余额均为 0，前端二次确认。
_Avoid_: 管理端改邀请人；有下级/订单/非零余额时硬删；仅硬删无软删

**代数奖励**:
每日分红的一种：按下级当日静态收益的 5% 计发，最多 10 代；直推 1 人解锁第 1 代，直推 n 人解锁前 n 代（n≤10）。结算静态时同步发放；FIFO 双记（出局累计+可提），无进行中单或超额部分作废。
_Avoid_: 直推奖（仅一代时）、级差奖；无单/超额仍进可提

**社区等级**:
按小区业绩对照 V1–V9：V1≥5千/10%，V2≥2万/20%，V3≥8万/30%，V4≥25万/35%，V5≥50万/40%，V6≥150万/45%，V7≥500万/50%，V8≥1000万/55%，V9≥2000万/60%；取满足门槛的最高档。
_Avoid_: VIP；自定义另一套门槛/比例（非本约定）

**社区基础奖**:
按**级差**计算：对下级树中每个有社区等级的节点，取该节点子树当日静态收益中**尚未被更深有等级节点占用的部分** × `(本人 V 级比例 − 该节点 V 级比例)`，再在本人小区范围内求和；比例差 ≤ 0 时该项为 0。本人与最近有等级下级之间的无等级段静态，按相对 **0%** 计，即 `该段静态 × 本人 V 比例`。与静态同日结算；FIFO 双记（出局累计+可提），无进行中单或超额部分作废。
_Avoid_: 按历史累计业绩每天全额乘比例；整棵子树静态不经去重直接乘；无等级段静态不参与级差；全局统一 calcBase × V 比例；仅按直推一条腿算级差；无单/超额仍进可提

**平级分红**:
沿邀请链的**上行同级奖**（对齐 new18new）：从当日有静态收益的用户往上走；无等级节点跳过；来源须在该祖先**小区**内（非最大腿；直推不足 2 人则无小区）；遇到更高社区等级则更新「上一档」；等级**低于**上一档则跳过（链上断档水位）；再遇到与上一档**同 V 级**且 ≥V3（`min_peer_level`）时，发放 `来源当日静态 × 平级比例`（默认 10%，`peer_pool_rate`）；单链最多 5 次。入账 FIFO 双记（出局累计+可提），无进行中单或超额部分作废。社区基础奖的「断档子树」规则不作用于平级（否则链上同级无法发平级）。
_Avoid_: 无单/超额仍进可提
_Avoid_: 直推同门基础奖建池均分（已废弃）；超越奖；全站同级共一池；从基础奖中划扣再建池

**业绩**:
某范围内**下级**用户的历史累计认购金额之和（含已出局；不因出局扣减）；**不含**该范围根节点本人的认购。
_Avoid_: 仅进行中认购合计；出局目标合计；把本人认购算进团队业绩

**小区**:
按直推拆腿：每个直推及其整棵下级树为一条腿；**小区业绩** = 去掉最大腿后其余各腿业绩之和。用于对照 V1–V9 定级。
_Avoid_: 整户合计直接定级（非本约定）；只看最大腿定级

**腿**:
某一个直推出发的整棵下级树；社区基础奖的断档按「断档节点及其子树」切分，不是整腿一刀切。
_Avoid_: 与小区混称；断档即整腿停发社区奖

**断档**:
对社区基础奖：若某节点社区等级 ≥ 领取人，则该节点及其下级的当日静态不再计入领取人的级差；同腿上断档点之上、或其他分支的静态仍可计。平级分红另按邀请链水位处理（见「平级分红」）。代数分红不受影响。
_Avoid_: 超越奖；整户一刀切停社区奖；断档时停代数；断档后整条腿停发
**停分红**:
管理端开关。开启后该用户仍可登录、仍拿本人静态与分红；但其当日静态**不再作为奖源**——上级不从他拿代数、社区基础奖、平级。
_Avoid_: 与软删（禁登录）混用；停本人静态结算
